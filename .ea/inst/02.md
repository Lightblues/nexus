# 修改总结
 P0: IPC Listener 内存泄漏修复

  preload/index.ts — 所有 ipcRenderer.on() 监听器现在返回 cleanup 函数：
  // Before: 无法取消
  onTick: (callback) => {
    ipcRenderer.on('pomodoro:tick', (_event, seconds) => callback(seconds))
  }

  // After: 返回取消函数
  onTick: (callback): (() => void) => {
    const handler = (_event, seconds) => callback(seconds)
    ipcRenderer.on('pomodoro:tick', handler)
    return () => ipcRenderer.removeListener('pomodoro:tick', handler)
  }

# 什么是 tap
  tap = 一个装着 formula/cask 定义的 GitHub 仓库。

  - 你运行 brew install --cask nexus 时，Homebrew 要去某个地方查"nexus 这个 cask
  的定义文件在哪、下载地址是什么、sha256 是多少"
  - 默认它只看官方 tap：homebrew/core (formula) 和 homebrew/cask (cask)
  - 自建 tap = 你自己开一个 GitHub repo，名字必须叫 homebrew-<something>，里面放 cask 定义文件
# 理解"签名"要解决什么
  macOS Gatekeeper 看到一个 app，会问三层问题：
  1. 有没有签名？ → 没签名的直接报"已损坏，移到废纸篓"（Sequoia 起更严格）
  2. 签名是谁的？ → Apple 颁发的 Developer ID 证书才被认可
  3. 有没有公证 (notarize)？ → Apple 扫过一遍确认没恶意代码

  ad-hoc 签名 = 自己给自己盖个章
  codesign --force --deep --sign - /path/to/Nexus.app

  为什么还是需要 xattr -dr com.apple.quarantine
  用户从浏览器下载 / brew 下载的 dmg 里的 app，macOS 会自动贴一个隔离标签 com.apple.quarantine。Gatekeeper
  只对贴了这个标签的 app 做严格检查。
# 比较
