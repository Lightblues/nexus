```sh
# 修改总结
 P0: IPC Listener 内存泄漏修复

  preload/index.ts — 所有 ipcRenderer.on() 监听器现在返回 cleanup 函数：
  // Before: 无法取消
  onTick: (callback) => {
    ipcRenderer.on('pomodoro:tick', (_event, seconds) => callback(seconds))
  }

  // After: 返回取消函数
  onTick: (callback): (() => void) => {
    const handler = (_event, seconds) => callback(seconds)
    ipcRenderer.on('pomodoro:tick', handler)
    return () => ipcRenderer.removeListener('pomodoro:tick', handler)
  }

  Renderer 组件 — PomodoroView、UploaderView、App.tsx 的 useEffect 现在正确调用 cleanup：
  useEffect(() => {
    const cleanupTick = window.api.pomodoro.onTick(...)
    const cleanupStatus = window.api.pomodoro.onStatus(...)
    return () => { cleanupTick(); cleanupStatus(); ... }
  }, [])

  P0: Pomodoro History 数据膨胀修复

  DataManager 新增自动归档机制：
  - 应用启动时自动将 90 天前的 session 移到年度归档文件 (~/.ea/nexus/archive/pomodoro-2025.json)
  - 归档时按 ID 去重，防止重复写入
  - 新增 getAllSessions() 方法，合并 active + archived 数据
  - StatsService 的 activity calendar 使用 getAllSessions() 获取完整历史

  PathManager 新增 archiveDir 属性，ensureDirectories() 自动创建 archive 目录。

  P1: 共享类型模块

  新文件 src/shared/types.ts — 所有跨进程共享的接口定义统一在此（约 250 行），包含：
  - Pomodoro (PomodoroStatus, SessionRecord, NextActionOption 等)
  - Stats (DailyStats, WeeklyStats, ActivityData 等)
  - Config (AppConfig, PomodoroConfig, TrackerConfig 等)
  - Tracker (WindowActivityRecord, DailyTrackerData 等)
  - Uploader (UploadRecord, CompressResult, ImageMeta 等)
  - Settings (ValidationResult, WriteResult)

  更新的文件（约 20 个）：main/core/, main/features/, preload/index.ts, renderer env.d.ts 和所有 feature
  views — 全部从 @shared/types 导入，消除了重复定义。

  构建配置：
  - electron.vite.config.ts — 三端都添加了 @shared alias
  - tsconfig.node.json / tsconfig.web.json — 添加 paths 和 include

  P1: Window 类型检测

  App.tsx — 移除了不可靠的 window.innerWidth > 500 判断，现在纯靠 URL hash 区分：
  - 有 #/stats, #/settings, #/tracker → MainWindow
  - 无 hash → Popup (dashboard)
```
